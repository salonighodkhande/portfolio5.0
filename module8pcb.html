Module 8 - IoT and PCB

Home

Day 1 – IoT Module

Objective: Learn about ESP8266 microcontroller and basic LED control programs.

Details: Explored ESP8266 pinout, specifications, and its importance in IoT. Installed Arduino IDE, configured it for ESP8266 boards, and installed necessary libraries. First programs included blinking onboard and external LEDs using GPIO pins. Students learned programming basics and hardware interfacing.

Day 2 – IoT Module

Objective: Understand Wi-Fi networking and create a simple web server.

Details: Learned Wi-Fi concepts (SSID, IP, password) and connection setup in ESP8266. Wrote code to connect ESP8266 to Wi-Fi and verified connection via Serial Monitor. Built a basic web server to control LEDs through a browser, introducing remote control using IoT.

Day 3 – IoT Module

Objective: Interface sensors and display real-time data on web pages.

Details: Reviewed Wi-Fi connection and web server from Day 2. Interfaced an ultrasonic sensor (HC-SR04) with ESP8266. Programs included real-time distance measurement printed in Serial Monitor and displaying live data on a hosted webpage for monitoring.

Day 4 – IoT Module

Objective: Design mobile app for sensor data monitoring and control.

Details: Discussed mini-project concept. Installed MIT App Inventor, designed a simple mobile app to display sensor values and control outputs. Integrated app with ESP8266 for two-way communication. Tested commands and data transfer successfully.

Day 5 – PCB Designing Module

Objective: Learn PCB basics and design AC to DC converter schematic.

Details: Designed AC to DC converter schematic using transformer, rectifier, capacitor filter, and voltage regulator. Learned PCB concepts: layers, traces, vias, footprints, and design rules. Discussed real-life applications and best design practices.

Day 6 – PCB Designing Module

Objective: Convert schematic to PCB layout and practice component placement.

Details: Converted AC to DC schematic into PCB layout using design software. Placed components strategically and routed tracks. Checked for Design Rule Check (DRC) errors and practiced independent layout designing under guidance.

Day 7 – PCB Designing Module

Objective: Design DC motor controller schematic and layout.

Details: Designed DC motor speed controller schematic using MOSFET/transistor and PWM. Converted schematic to PCB layout, routed connections, and ensured design integrity. Discussed real-world implementation, simulation tips, and cleared design doubts.
